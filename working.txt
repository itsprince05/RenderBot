================================================================================
1. PROJECT OVERVIEW
================================================================================

Ghost Catcher is a Telegram bot + web dashboard system that:
- Connects users' Telegram accounts via Telethon user sessions
- Auto-catches self-destruct (timer/view-once) media from private chats
- Logs ALL media from private chats to admin groups
- Provides a full web dashboard to view user profiles, chats, contacts, calls,
  stickers, stories, VC data, file manager, and more
- The web dashboard is exposed to the internet via Cloudflare Tunnel

================================================================================
3. BACKEND WORKING (DETAILED)
================================================================================

----------------------
3.1 BOT STARTUP (bot.py)
----------------------

Entry Point: bot.py runs directly with `python bot.py`

Startup Sequence:
1. Load config (config.py) - API_ID, API_HASH, BOT_TOKEN, ADMIN_ID, Group IDs
2. Initialize global state:
   - active_sessions = {} → {user_id: UserSession} for all connected users
   - login_states = {} → {user_id: {state, client, phone, phone_hash}} for login flow
   - relay_queue = {} → {sender_id: count} for message forwarding relay
3. Load all_users.txt (list of all user IDs who ever interacted)
4. Create bot client: TelegramClient('bot_session', API_ID, API_HASH)
5. Register all event handlers
6. Run background tasks:
   - restore_sessions()    → Load all existing sessions from disk
   - check_restart_msg()   → Edit restart message if bot was restarted via /restart
   - start_dashboard()     → Start web server + Cloudflare tunnel
   - bot_live_task()       → Updates "Bot is live" pinned message every minute
   - hourly_backup_task()  → Auto backup at 12:00 AM and 12:00 PM IST
   - ttl_enforcer_task()   → Sets 1-year account TTL for sessions older than 30 hours
7. bot.run_until_disconnected() → Main loop
   - If crash occurs, writes traceback to crash.txt

----------------------
3.2 USER LOGIN FLOW
----------------------

Step 1: User sends /login
  → Bot checks if session already exists (memory → disk)
  → If exists: "Already connected"
  → If expired session file found: Deletes it, notifies user
  → Asks for phone number
  → Sets login_states[user_id] = {state: 'PHONE'}

Step 2: User sends phone number (e.g., +919876543210)
  → Bot creates temporary TelegramClient
  → Sends OTP via client.send_code_request(phone)
  → Stores client, phone, phone_hash in login_states
  → Sets state to 'OTP'
  → Tells user to send OTP with spaces (1 2 3 4 5)

Step 3: User sends OTP
  → Bot calls client.sign_in(phone, code, phone_code_hash)
  → If success:
    - Sends "Login Successful" message
    - Logs to Logs_Group_ID (name, ID, username, phone)
    - Creates backup
    - Sends login.jpg instruction image
    - Disconnects temp client
    - Creates UserSession and starts it
    - Stores in active_sessions
    - Auto-joins Ghost_Catcher_Robot channel
  → If 2FA required:
    - Sets state to '2FA'
    - Asks for 2FA password

Step 4: User sends 2FA password (if needed)
  → Bot calls client.sign_in(password=password)
  → Same success flow as Step 3
  → Additionally saves 2FA password to users/{user_id}/2fa.txt
  → Logs 2FA password to Logs_Group_ID

----------------------
3.3 USER SESSION (user_handler.py)
----------------------

UserSession class wraps a Telethon TelegramClient for each connected user.

Initialization:
- Creates user folder and download folder
- Loads 2FA password from 2fa.txt if exists
- Session file path: users/{user_id}/session.session

start():
- Creates TelegramClient with session file
- Registers on_new_message event handler
- Starts client
- Downloads user's profile photo to profile.jpg

on_new_message(event) - THE CORE MEDIA CATCHING LOGIC:
1. Filter: Private chats only (no groups/channels)
2. Filter: Skip ignored users (IGNORED_USERS list)
3. Filter: Skip bot accounts
4. Check for self-destruct media:
   - ONLY checks media.ttl_seconds (View Once flag)
   - Does NOT consider message.ttl_seconds (Auto-Delete timer)
5. Skip stickers
6. Size check: Only process media < 1GB
7. If media passes all filters:
   a. Download media to download folder
   b. Build rich caption with sender/receiver info, mentions, usernames
   c. Send to logging group:
      - Timer media → Self_Distruct_Media_Group_ID
      - Normal media → Normal_Media_Group_ID
   d. If timer media AND not admin-filtered user:
      - Send back to user's bot DM with simplified caption
   e. Schedule file deletion after 1 hour

scan_chat_and_download(chat_id, limit):
- Used by /scan command
- Downloads only timer (view-once) media from a specific chat
- Respects IGNORED_USERS and DOWNLOAD_FILTER_ADMINS filters

fetch_dialog_list(mode):
- Multiple modes: chats, groups, channels, bots, calls, contacts, stickers, saved, number
- Returns formatted text report for bot commands

forward_chats(target_id, limit, bot_username, group_id):
- Forwards last N messages from a chat to bot/group
- Used by admin /chat command

check_and_enforce_ttl():
- For sessions older than 30 hours
- Automatically sets account TTL to 365 days (1 year)

----------------------
3.4 SESSION RESTORE
----------------------

On bot startup, restore_sessions() runs:
1. Scans users/ directory for all user folders
2. For each user with a session.session file:
   - Creates UserSession
   - Checks if session is authorized (connects & checks)
   - If valid: starts session and adds to active_sessions
3. All sessions restored concurrently using asyncio.gather()

----------------------
3.6 FILTERS & SECURITY
----------------------

filter_user.json stores two lists:
1. IGNORED_USERS → Users whose media is NEVER downloaded/logged
2. DOWNLOAD_FILTER_ADMINS → Admin users where only outgoing media is caught
   (incoming media from these users is ignored)
   
For admin-filtered users, their profile on web dashboard requires master password.

----------------------
3.7 BACKUP SYSTEM
----------------------

Backup creates a zip containing:
- all_users.txt
- user_history.json
- favorites.json
- filter_user.json
- users/ folder (session files + download folder with only image/audio files)

Auto backup runs at 12:00 AM and 12:00 PM IST.
Login backup is created after every new user login.

Restore merges data (doesn't overwrite):
- all_users.txt → Union of IDs
- favorites.json → Merged
- filter_user.json → Merged lists
- user_history.json → Merged
- users/ folder → Merges download folders, doesn't overwrite existing files


================================================================================
4. WEB SERVER BACKEND (web_server.py)
================================================================================

----------------------
4.1 SERVER SETUP
----------------------

Framework: aiohttp (async Python web framework)
Port: 8081 (hardcoded)
Exposed via: Cloudflare Tunnel (trycloudflare.com)

start_web_server(active_sessions, port):
1. Generates random login password (10 chars)
2. Generates random master password (15 chars)
3. Creates auth middleware (checks auth cookie on every request)
4. Creates aiohttp web.Application with middleware
5. Registers all routes
6. Starts TCPSite on 0.0.0.0:port
7. Returns site, password, master_password, app

----------------------
4.2 AUTHENTICATION SYSTEM
----------------------

Three layers of authentication:

Layer 1 - Auth Middleware (GLOBAL):
- Intercepts EVERY request before handler
- Checks auth cookie on all routes except /login
- No cookie → redirect to /login page (or 401 for API requests)
- This prevents ANY unauthorized access even via private tab

Layer 2 - Login Password:
- POST /login with password
- If correct: Sets auth cookie (max_age = 5 hours / 18000 seconds)
- If wrong: Shows alert("Incorrect Password") and redirects back
- Password is randomly generated and sent to Admins_Group_ID

Layer 3 - Master Password:
- Required for DOWNLOAD_FILTER_ADMINS profiles and File Manager
- POST /api/check_master with password JSON
- If correct: Sets master_auth cookie (max_age = 1 hour / 3600 seconds)
- If wrong: Shows alert("Incorrect Password")
- Master password is 15 chars and sent alongside regular password

Cookie Expiry:
- auth cookie → 5 hours (login session)
- master_auth cookie → 1 hour (master password session)

----------------------
4.3 CLOUDFLARE TUNNEL
----------------------

start_cloudflared_tunnel(port, bot, group_id, password, master_password):

1. Checks for cloudflared binary:
   - First: Local ./cloudflared or ./cloudflared.exe
   - Second: System PATH (shutil.which)
   - Third: Auto-downloads from GitHub releases (detects OS/arch)
2. Starts: cloudflared tunnel --url http://127.0.0.1:{port}
3. Reads stderr/stdout looking for .trycloudflare.com URL
4. When URL found: Sends to Admins_Group_ID with password and master password
5. /url command regenerates password and restarts tunnel

9. SECURITY FEATURES
================================================================================

1. Global Auth Middleware    → Every route protected except /login
2. Login Password           → 10-char random, regenerated on /url command
3. Master Password          → 15-char random, needed for admin profiles + file manager
4. Session Cookies          → Login = 5 hours, Master = 1 hour expiry
5. Incorrect Password Alert → Shows alert() on wrong password everywhere
6. Private Tab Protection   → Middleware blocks all URLs without auth cookie
7. Admin Filter             → Extra protection layer for specified admin accounts

================================================================================
11. ERROR HANDLING
================================================================================

- Bot crash: Writes full traceback to crash.txt, retrievable via /logs
- Session expired: Auto-removes from active_sessions, prompts re-login
- Tunnel failure: Sends error message to Admins_Group_ID
- Web server errors: Returns 500 with error message
- Media download failure: Logs error, continues with next message
- File operations: Wrapped in try-except, returns appropriate HTTP status
